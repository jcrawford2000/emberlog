# Web Package Docs Policy

This folder is for package-local operational documentation and assets.

## Source of Truth

Normative architecture and contracts are in root `/docs`.

Canonical references:

- `/docs/PLATFORM_VISION_v0.2.md`
- `/docs/DEPLOYMENT_MODEL_v0.1.md`
- `/docs/EVENT_MODEL_v0.2.md`
- `/docs/API_CONTRACT_v0.1.md`
- `/docs/WEB_ARCHITECTURE_v0.1.md`
- `/docs/DEVELOPMENT.md`

## Allowed Content Here

- UI screenshots
- Frontend runbooks and troubleshooting notes
- Package-local operational guidance

## Not Allowed Here

- Parallel API contracts
- Alternate event model definitions
- Duplicate platform architecture requirements
- Competing current-state specs for the platform

If behavior or contracts change, update root `/docs` first.
