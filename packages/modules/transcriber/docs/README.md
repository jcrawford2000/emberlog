# Transcriber Docs Policy

This folder is for module-local operational documentation.

## Source of Truth

Normative platform architecture and contracts are defined in root `/docs`.

Canonical references:

- `/docs/PLATFORM_VISION_v0.2.md`
- `/docs/DEPLOYMENT_MODEL_v0.1.md`
- `/docs/EVENT_MODEL_v0.2.md`
- `/docs/API_CONTRACT_v0.1.md`
- `/docs/DEVELOPMENT.md`

## Allowed Content

- Local runbooks and quickstarts
- Integration notes for current module behavior
- Operational troubleshooting

## Not Allowed Content

- Parallel platform requirements/architecture specs
- Alternate canonical API/event contract definitions
- Competing platform current-state documents

If behavior or contracts change, update root `/docs` first, then module docs.
